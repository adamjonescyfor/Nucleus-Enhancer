# Roadmap — CYFOR Nucleus Enhancer

Implementable backlog beyond v3.0. Each item is specced so a developer (or future AI session) can build it without re-deriving context. Ordered within each wave by impact ÷ effort. **Hard rules that always apply:** Salesforce field API names are describe-discovered (never hardcode); OAuth secrets stay in Cloudflare; `report/disclosure-report.js` + `styles/case-report.css` are the CTO's (don't edit); the MG22 feature is Mitul's (flagged off via `MG22_ENABLED` in `content/case-report.js`).

### Exhibit template sheets (PDF) — M/L (content-script button + true-scale PDF)
**Original ask:** "Is there any way Nucleus could generate exhibit template sheets? Ideally as a PDF so we can easily print from acquisition PCs?" — i.e. a one-click **PDF** an examiner can print straight from an acquisition PC.
A button injected on an **Exhibit** record (under its Case), alongside the existing injected actions (case report / photo download). Clicking it generates a **printable pre-imaging sheet** to lay beside the exhibit when photographing it before acquisition: a **calibrated ruler** (true-to-life scale) plus the **case reference and exhibit reference** auto-filled from the record (describe-discovered — never hardcode API names; read via the record id → SOQL like `background/sf-cases.js`, or scrape the page). Output is a PDF specifically so it prints reliably from locked-down acquisition machines.
- **The hard requirement is true physical scale** — the ruler must measure correctly off the printer, so the output must carry absolute dimensions and be printed at **100% / Actual size** (not "fit to page").
- **Approach:** draw the PDF directly in the extension with a bundled PDF library (pdf-lib / jsPDF) in absolute units — page = a real paper size (A4), ruler length set in points (1 pt = 1/72"; 1 mm ≈ 2.835 pt), so a 150 mm ruler renders as exactly 150 mm regardless of viewer. Print a **calibration bar** on the sheet ("this line should measure exactly 100 mm — verify after printing") so the examiner can confirm the printer didn't rescale. (An HTML print-view with CSS `mm` + `@page { size: A4; margin: 0 }` is a lighter alternative, but browser print scaling is less reliable than a unit-exact PDF.)
- **"Same scaling as the docx":** keep the calibrated **.docx** as the design source of truth for the ruler; the generated PDF must reproduce its exact scale — validate by measuring a printout of each against a physical ruler. Bundle the ruler artwork + sheet layout with the extension; only the case/exhibit refs come from Nucleus (no new Salesforce object needed).
